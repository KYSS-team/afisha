# Система электронной афиши

Проект описывает веб-систему для управления событиями с тремя модулями: авторизация, события и администрирование. Ниже приведены ключевые возможности и принципиальные решения архитектуры.

## Краткий обзор модулей
- **Авторизация**: регистрация с подтверждением почты, вход, восстановление пароля, уведомления о смене пароля.
- **События**: каталог с вкладками «Активные», «Мои события», «Прошедшие», карточка события с подтверждением/отменой участия, автоматическое обновление статусов.
- **Администрирование**: управление пользователями и событиями, создание/редактирование событий, фильтры, сброс пароля пользователю, экспорт участников.

## Технологии
- Backend: Kotlin + Spring Boot, база данных PostgreSQL.
- Frontend: Next.js + TypeScript.
- Интеграции: SMTP (через адаптер) для всех почтовых уведомлений.

## Что в репозитории
- `docs/architecture.md` — подробная архитектура, схемы данных и API, почтовые уведомления, сценарии UI.

## Как развивать дальше
1. Развернуть сервис отправки почты (MailHog локально или SMTP в `.env`).
2. Инициализировать backend-проект Spring Boot (Gradle Kotlin DSL) согласно структуре из `docs/architecture.md`.
3. Инициализировать фронтенд Next.js с маршрутизатором App Router и Tailwind/Chakra UI; реализовать страницы аутентификации и список событий по схемам из документации.
4. Настроить CI: линтеры, unit/integration тесты, сборка Docker для двух сервисов.
