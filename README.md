# Электронная афиша

Многостраничное приложение для регистрации пользователей, публикации и модерации событий. Репозиторий включает backend на Spring Boot/Kotlin и frontend на Next.js/TypeScript; используется PostgreSQL и SMTP (MailHog) для работы с почтой. Архитектурные детали и доменная модель описаны в `docs/architecture.md`.

## Структура репозитория
- `backend` — REST API, авторизация по JWT, обработка событий и участие пользователей.
- `frontend` — интерфейс для участников и администраторов (Next.js + Tailwind CSS).
- `docker-compose.yml` — инфраструктурные контейнеры PostgreSQL и MailHog.
- `docs/` — дополнительная документация по архитектуре.

## Требования
- Docker и Docker Compose — для запуска PostgreSQL и почтового тестового сервера.
- Java 21 и Gradle 8+ — для сборки и запуска backend.
- Node.js 18+ и npm — для фронтенда.

## Быстрый старт
1. Поднимите инфраструктуру:
   ```bash
   docker-compose up -d
   ```
   Это создаст локальные контейнеры `postgres:15` (порт 5432) и MailHog (SMTP: 1025, веб-интерфейс: 8025).
2. Настройте backend: следуйте инструкциям из `backend/README.md` (настройка БД и SMTP, запуск приложения на 8080).
3. Настройте frontend: шаги описаны в `frontend/README.md` (указание базового URL API и запуск dev-сервера на 3000).
4. Откройте http://localhost:3000 чтобы попасть в интерфейс. API по умолчанию слушает http://localhost:8080.

## Краткое описание возможностей
- Регистрация и подтверждение e-mail, авторизация и обновление токенов, восстановление пароля через почту.
- Управление событиями: создание карточек с описанием, загрузкой изображений, подтверждением/отменой участия, оценками и экспортом участников.
- Админ-функции: модерация событий, работа со статусами, выдача ролей, просмотр пользователей.
- Интерфейс с вкладками событий (мои, активные, прошедшие), карточками и быстрыми действиями, страницами входа/регистрации и админ-панелью.
